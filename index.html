🏷️ HTML — HyperText Markup Language
HTML 01 — What is HTML? Structure of a Web Page
▼
Definition
HTML (HyperText Markup Language) is the skeleton of every web page. It tells the browser: "here is a heading", "here is a paragraph", "here is an image". Think of HTML as the bones, CSS as the skin/clothes, and JavaScript as the muscles that make it move.
HTML
<!-- Basic structure of every HTML file -->
<!DOCTYPE html>               <!-- Tells browser: use HTML5 -->
<html lang="en">               <!-- Root element, lang = language -->
  <head>                        <!-- Invisible metadata zone -->
    <meta charset="UTF-8"/>     <!-- Accept all characters (Arabic ✓) -->
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0"/>
    <title>Page Title</title>
    <link rel="stylesheet" href="style.css"/>
  </head>
  <body>                        <!-- Everything VISIBLE goes here -->
    <h1>Hello World!</h1>
    <p>This is a paragraph.</p>
  </body>
</html>
⚠️ EXAM: The <viewport> meta tag is essential for responsive design. Without it, mobile browsers zoom out and show the desktop version. Always include it!

⚠️ EXAM: <div> and <span> are NOT semantic — they are "generic containers". <header>, <section>, <article>, <footer>, <nav>, <aside> ARE semantic. Also: <maincontent>, <box>, <sidebar>, <sectioned> — these DO NOT EXIST!
Why use semantic HTML?
📌 1. Improves accessibility (screen readers know what each part is). 2. Helps SEO (Google understands your content better). 3. Makes code more readable for other developers.

The href attribute defines the URL. <a> is an inline element.
HTML

🎨 CSS — Cascading Style Sheets
CSS 01 — CSS Basics: Selectors, Properties, Values
▼
A CSS rule has 3 parts: selector (who to target), property (what to change), value (the new value).
CSS
/* selector { property: value; } */
h2 { color: darkblue; }
/* ^selector  ^property ^value */

/* Multiple properties in one rule */
p {
  color: red;
  font-size: 16px;
  line-height: 1.6;
}

/* Grouping: apply same style to multiple selectors */
h1, h2, h3 { color: blue; }

CSS 02 — Specificity & The Cascade (🔥 Exam Favorite!)
▼
When two rules fight over the same element, the browser uses specificity to decide which one wins. Higher specificity = wins.
!important — overrides EVERYTHING
∞ WINS
Inline style: style="color:red"
1,0,0,0
ID selector: #header
0,1,0,0
Class .card / Attribute [type] / Pseudo-class :hover
0,0,1,0
Type selector p, h1 / Pseudo-element ::before
0,0,0,1
⚠️ If specificity is EQUAL, the LAST rule in the file wins (cascade). That's the "C" in CSS.
3 Ways to Include CSS
HTML
<!-- 1. EXTERNAL (Best Practice!) -->
<link rel="stylesheet" href="style.css"/>

<!-- 2. INTERNAL (inside <head>) -->
<style>
  body { background: white; }
</style>

<!-- 3. INLINE (on the element itself) -->
<h1 style="color: red; font-size: 2rem;">Hello</h1>

<!-- 4. @import inside CSS file -->
/* @import url("other.css"); */
⚠️ EXAM: Valid CSS inclusion methods: inline style, <link> external, <style> internal, @import. INVALID: embedding CSS in <script> tag!
CSS 03 — The CSS Box Model (🔥 Always on Exam!)
▼
Every HTML element is a rectangular box with 4 layers: Content → Padding → Border → Margin
MARGIN (transparent space outside)
BORDER
PADDING (space inside border)
CONTENT (text, images)
CSS
.box {
  /* CONTENT */
  width: 300px;
  height: 200px;

  /* PADDING — space between content and border */
  padding: 20px;            /* all 4 sides */
  padding: 10px 20px;       /* top/bottom left/right */
  padding-top: 10px;        /* one side only */

  /* BORDER */
  border: 2px solid blue;   /* width style color */

  /* MARGIN — space OUTSIDE the border */
  margin: auto;              /* centers block element */
  margin: 0 auto;           /* top/bottom=0, left/right=auto */

  /* IMPORTANT: box-sizing */
  box-sizing: border-box;   /* width includes padding+border */
}
⚠️ EXAM Q21 (2024): To reduce space between content and border → use padding. Margin is for OUTSIDE the border. Box model properties: padding, margin, border. NOT: color, text-align!

float property
float is used to wrap text around images. Floated elements are removed from the normal flow. Use clear: both after floats to avoid layout issues.
CSS
.element {
  float: left;          /* text wraps around right side */
  margin-right: 20px;  /* space between float and text */
}
/* After floated elements: */
.clearfix { clear: both; }
CSS 05 — Flexbox (🔥 MOST Important CSS Topic!)
▼
Flexbox is a 1-dimensional layout system (row OR column). Add display: flex to the parent container, and all direct children become flex items.
CSS
/* PARENT CONTAINER properties */
.container {
  display: flex;

  /* Direction */
  flex-direction: row;          /* row | column | row-reverse | column-reverse */

  /* Main axis alignment (horizontal if row) */
  justify-content: space-between; /* flex-start | center | space-around | space-evenly */

  /* Cross axis alignment (vertical if row) */
  align-items: center;           /* flex-start | flex-end | stretch | baseline */

  /* Wrapping */
  flex-wrap: wrap;               /* nowrap | wrap | wrap-reverse */

  /* Gap between items */
  gap: 1rem;
}

/* CHILD ITEMS properties */
.item {
  flex-grow: 1;     /* grow to fill available space */
  flex-shrink: 1;   /* shrink if needed */
  flex-basis: 200px; /* starting size */
  align-self: flex-end; /* override align-items for this item */
  order: 2;           /* change visual order */
}
Item 1
Item 2
Item 3
Item 4
⚠️ EXAM Q3 (2025): ✅ Flexbox is one-dimensional. ✅ align-items aligns along the cross axis. ❌ justify-content does NOT control vertical alignment (it's main axis). ❌ Flexbox is NOT only for inline elements.
✅ To center items both horizontally AND vertically: justify-content: center; align-items: center; — this was Exam 2024 Q22!
CSS 06 — CSS Grid, Variables, Transitions
▼
CSS Grid is 2-dimensional (rows AND columns simultaneously). Better for page-level layouts. Use Flexbox for components, Grid for full layouts.
CSS
.grid {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;    /* 3 columns */
  grid-template-columns: repeat(3, 1fr); /* same, cleaner */
  gap: 1rem;

  /* Responsive without media queries! */
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
}

/* CSS Variables (Custom Properties) */
:root {
  --primary-color: #2563eb;
  --spacing: 1rem;
}
.card {
  background: var(--primary-color);  /* use with var() */
  padding: var(--spacing);
}

/* Transitions */
.btn {
  background: blue;
  transition: all 0.3s ease;  /* property duration timing */
}
.btn:hover { background: darkblue; }

JS 02 — DOM Manipulation (🔥 Most Important JS Topic!)
▼
The DOM (Document Object Model) is JavaScript's way of accessing and changing HTML. Think of the HTML page as a tree, and JavaScript can grab any branch and change it.
Selecting Elements
JS
// By ID — returns ONE element
const title = document.getElementById('main-title');

// By class name — returns HTMLCollection (multiple)
const cards = document.getElementsByClassName('card');

// By tag name — returns all <p> tags
const paras = document.getElementsByTagName('p');

// querySelector — CSS selector, returns FIRST match
const btn = document.querySelector('.btn');     // class
const hd = document.querySelector('#header');   // id

// querySelectorAll — returns ALL matches
const allBtns = document.querySelectorAll('.btn');
⚠️ EXAM Q34 (2024): To get elements by class: document.getElementsByClassName('name'). FAKE methods: document.modifyElement(), window.selectElement() — these don't exist!
Changing Content
JS
const el = document.getElementById('box');

// Change HTML content (can include tags)
el.innerHTML = "<strong>Hello</strong>";  // ✅ Valid

// Change text only (no tags)
el.textContent = "Hello";                  // ✅ Valid

// For input fields
el.value = "Hello";                        // ✅ For inputs

// ❌ These DON'T EXIST in JS:
// el.setText("Hello")   ← FAKE
// document.write() — exists but avoid (replaces whole page)
Changing Styles
JS
const el = document.getElementById('box');

// Method 1: style property (camelCase!)
el.style.color = "red";
el.style.backgroundColor = "blue";  // NOT background-color!
el.style.fontSize = "20px";

// Method 2: setAttribute
el.setAttribute("style", "color:red");

// Method 3: classList (best for toggles!)
el.classList.add("active");      // add class
el.classList.remove("active");   // remove class
el.classList.toggle("active");   // toggle on/off
el.classList.contains("active"); // check if has class

// ❌ FAKE methods (don't exist):
// el.changeStyle("color","red")  ← FAKE
// el.css("color","red")           ← FAKE (that's jQuery!)
JS 03 — Events: onclick, addEventListener, onchange
▼
Events are actions that happen: user clicks, types, submits a form. JavaScript listens and reacts.
HTML + JS
<!-- Method 1: HTML attribute (onclick) -->
<button onclick="sayHello()">Click me</button>
<input onchange="doSomething()" type="text"/>
<form onsubmit="handleSubmit()"></form>

// Method 2: addEventListener (best practice!)
const btn = document.getElementById('myBtn');
btn.addEventListener('click', function() {
  alert('Button clicked!');
});

// Arrow function version
btn.addEventListener('click', () => {
  console.log('clicked');
});
⚠️ EXAM Q10 (2025): Valid event triggers: onclick attribute, addEventListener("click",...), onchange on inputs, onsubmit on forms. INVALID: document.onhover — there's no "onhover" event!

Array Methods
JS
const fruits = ['apple', 'banana'];

const letters = new Set(['a', 'b', 'c']);
for (let letter of letters) { // ✅ "of" for values
  console.log(letter);
}
for (let i in letters) {   // ❌ "in" gives indexes/keys not values

}

}

do {

} while (condition);

📱 Responsive Design & Media Queries
RESP 01 — What is Responsive Design?
▼
Responsive design means your website looks good on ALL screen sizes — from a tiny phone to a large desktop monitor. It uses flexible layouts, images, and CSS media queries.
📌 The 3 pillars of responsive design: 1. Fluid grids (% widths instead of px). 2. Flexible images. 3. Media queries.
The Viewport Meta Tag — MANDATORY!
HTML
<!-- WITHOUT this, mobile browsers show desktop size zoomed out -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0"/>

<!-- width=device-width: page width = screen width -->
<!-- initial-scale=1.0: don't zoom in/out by default -->
⚠️ EXAM Q17 (2025): The viewport meta tag: ✅ controls how page is scaled on mobile, ✅ allows responsive layout, ✅ exact syntax as above. ❌ NOT only for desktop! ❌ Without it, most mobiles show the page in desktop mode (zoomed out).
RESP 02 — Media Queries: @media
▼
Media queries apply CSS rules ONLY when specific conditions are met (screen size, device type, etc.).
CSS
/* Mobile-first: write default styles for SMALL screens */
.container {
  flex-direction: column;  /* mobile: stack vertically */
}

/* Tablet: apply when screen is 768px or WIDER */
@media (min-width: 768px) {
  .container { flex-direction: row; }
}

/* Desktop */
@media (min-width: 1024px) {
  .sidebar { display: block; }
}

/* Dark mode preference */
@media (prefers-color-scheme: dark) {
  :root { --bg: #0a0f14; }
}

/* You can also combine conditions */
@media (min-width: 600px) and (max-width: 1024px) {
  /* tablet ONLY */
}

⚠️ EXAM Q19 (2025): Valid media query features: max-width, min-height, device-pixel-ratio, aspect-ratio. Valid: prefers-color-scheme. NOTE: screen-orientation is valid but usually written as orientation.
